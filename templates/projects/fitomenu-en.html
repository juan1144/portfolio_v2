FitoMenu

Restaurant management system with digital menu, orders and kitchen panel.

About the project

FitoMenu is a web application developed for the local restaurant Pollo Fito, with the goal of modernizing and digitizing its customer service and internal management process. The system allows customers to access the menu via a QR code from their mobile device, view available products and place orders in real time. Internally, kitchen staff receive these orders through a web interface to manage them more efficiently and orderly.

This project arose from the need to optimize service times, reduce manual errors and improve the customer experience. The solution was designed considering the particularities of a small business, allowing for gradual, intuitive and sustainable implementation over time.

Technologies used

Django Bootstrap PostgreSQL HTMX Docker Git Github

Key features

  • Interactive Digital Menu: Responsive web interface accessible via QR code, where customers can explore products organized by categories, view images, descriptions and prices.
  • Order Placement and Management: Complete flow for product selection and real-time order generation. Orders are automatically sent to the kitchen panel for validation and processing.
  • Kitchen Dashboard (Operational Panel): Module designed for kitchen staff, allowing them to view incoming orders, confirm or reject orders, and organize preparation in real time. Offers an optimized view focused on operational agility.
  • Administrative Dashboard: View intended for restaurant administrators showing key statistics like best-selling products in different time ranges. Facilitates data-driven decision making.
  • Product and Menu Management: Complete product CRUD with availability control, categories, images, descriptions and prices. Includes validations and Bootstrap-styled forms.

Challenges and solutions

Real-time Order Synchronization

Challenge: Achieve smooth communication between the customer menu and the kitchen system without delays.
Solution: An asynchronous order flow was designed with clear status logic and immediate updates, using well-optimized views to ensure an uninterrupted experience.

Strategic Information Presentation

Challenge: Incorporate useful metrics for decision-making in a user-friendly interface for non-technical users.
Solution: An administrative dashboard was developed showing best-selling products by period, using SQL query aggregations and clear visual design with Bootstrap.

Project Modularization

Challenge: Keep the code scalable and maintainable for future expansions.
Solution: The project was structured respecting separation of concerns: models, forms, views, templates and business logic, following best practices recommended by Django's official documentation.

Training and Usability for Restaurant Staff

Challenge: Ensure technology adoption by non-technical staff.
Solution: A simple Bootstrap-based interface was implemented, accompanied by a user manual and in-person training session to ensure proper system use.

Mobile Adaptability and Performance

Challenge: Guarantee a fast and functional experience on mobile devices with limited connection.
Solution: Light frontend loading was prioritized, mobile-first design principles were applied, and each view was optimized to minimize unnecessary queries and loading.
